# API Designer Command

Design or review an API using the api-designer subagent.

## Task Description

$ARGUMENTS

## Instructions

1. Analyze the API design requirements:
   - Business requirements and use cases
   - Client needs and constraints
   - Integration patterns
   - Performance requirements

2. Delegate to the `api-designer` subagent to:
   - Design the API following REST/GraphQL principles
   - Create OpenAPI 3.1 specification
   - Define consistent naming conventions
   - Design error responses and handling
   - Configure authentication and authorization
   - Implement pagination and filtering
   - Plan rate limiting strategy
   - Ensure backward compatibility

3. The api-designer will analyze according to the checklist:
   - RESTful principles properly applied
   - OpenAPI specification complete
   - Consistent naming conventions
   - Comprehensive error responses
   - Pagination implemented correctly
   - Rate limiting configured
   - Authentication patterns defined
   - Backward compatibility ensured

4. Provide output including:
   - API design overview
   - Resource definitions
   - Endpoint specifications
   - Request/response schemas
   - Authentication flows
   - Error handling strategy
   - Developer experience considerations
   - Integration recommendations

## Output Format

Output the API design in Japanese, following the api-designer subagent's response format.
