# Architect Reviewer Command

Review system architecture using the architect-reviewer subagent.

## Architecture Review Topic

$ARGUMENTS

## Instructions

1. Understand the architecture review scope:
   - System purpose and design goals
   - Current architecture or design documents
   - Technical requirements and constraints
   - Scalability and performance needs
   - Team capabilities and expertise

2. Delegate to the `architect-reviewer` subagent to:
   - Analyze architectural decisions
   - Evaluate design patterns
   - Assess scalability potential
   - Review technology choices
   - Identify technical debt
   - Check security considerations
   - Evaluate maintainability
   - Plan evolution strategy

3. The architect-reviewer will assess:
   - Design patterns appropriateness
   - Scalability requirements fulfillment
   - Technology choices justification
   - Integration patterns soundness
   - Security architecture robustness
   - Performance architecture adequacy
   - Technical debt management
   - Evolution path clarity

4. Provide architectural review including:
   - Design pattern evaluation
   - Scalability analysis
   - Technology stack assessment
   - Risk identification
   - Technical debt items
   - Modernization recommendations
   - Evolution roadmap
   - Strategic guidance

## Output Format

Output the architecture review in Japanese, following the architect-reviewer subagent's response format.
